# Information Architecture

## Purpose

The Information Architecture (IA) defines how information is organized, connected, and discovered within Neuraverse. It ensures users can easily navigate the platform while AI can structure knowledge into an evolving Living Knowledge Network.

## IA Principles

- Conversations are the primary knowledge objects.
- Knowledge is organized by Domains, not folders.
- Perspectives, Evidence, Debates, and AI Insights enrich conversations.
- Every object can be linked to build a knowledge graph.
- Navigation should minimize cognitive load.

## Core Information Hierarchy

1. Home
2. Discover
3. Domains
4. Conversations
5. Experts
6. Knowledge Network
7. AI Assistant
8. User Workspace

## Knowledge Object Relationships

Domain → Conversation → Perspective → Evidence → Debate → AI Summary → Knowledge Network

## Design Goals

- Easy discovery
- Scalable organization
- Transparent relationships
- AI-assisted navigation
- Long-term knowledge evolution
